# UI/UX 개선 — 2차 라운드

## 사용자 질문에 대한 답변

### Q1. 썸네일 카드 Enter/Space 열기, 필요한가?
**필요합니다.** 현재 카드는 `<button>`이라 키보드 사용자(또는 Tab 네비게이션)에게 유일한 활성화 수단이 키보드입니다. 브라우저 기본 `<button>`은 Enter/Space를 자동 처리하므로 별도 onKeyDown은 사실상 중복이지만 — 현재 구현(`handleKeyOpen`)도 무해합니다. **결론: 그대로 유지** (제거 시 일부 스크린리더/포커스 트랩 환경에서 동작이 깨질 수 있음).

### Q2. 상단 필터 버튼 디자인 검토
현재 디자인의 강점과 약점:

**강점**
- 부서별 컬러 닷 + glow → 시각적 식별성 좋음
- active 상태 그라데이션 → 선택 상태 명확
- count 표시 → 정보 밀도 높음

**약점**
1. **모든 버튼이 항상 같은 두께** → 시각적 위계 없음 ("부서"와 "상태" 라벨이 버튼과 동일한 15px). 라벨이 좀 더 작거나 흐려야 그룹 구분이 됨.
2. **버튼 크기가 큼** (`px-4 py-2 text-[15px]`) → 5+4=9개 버튼 + 검색창이 한 줄에 빠듯. 1920px에서도 검색창이 줄바꿈 되는 원인.
3. **count 가독성** → `(5)`가 본문과 같은 굵기·크기라 시각 노이즈.
4. **Divider(`h-6 w-px`)가 너무 얇음** → 부서/상태 그룹 분리감 약함.
5. **"전체" 버튼만 컬러 닷이 없음** → 다른 버튼과 정렬이 미세하게 어긋남(좌측 패딩 시각 차이).

→ **개선안 적용** (아래 #5에서 구체화).

## 수정 항목

### #1. FilterBar 줄바꿈 방지 (긴급)
**원인**: `전체 초기화` 버튼 추가로 우측 영역이 넓어져 좌측 부서/상태 그룹이 한 줄에 못 들어가면서 wrap 발생 → 행 두께가 굵어짐.

**수정**:
- 최상위 컨테이너에 `flex-nowrap` 강제
- 좌측 그룹(`gap-8`)을 `gap-6`로 축소, 내부 `gap-3`을 `gap-2`로 축소
- 부서/상태 버튼 패딩 `px-4 py-2` → `px-3 py-1.5`, 폰트 `15px` → `14px`
- 라벨("부서", "상태") 폰트 `15px` → `13px`, 색상 더 흐리게 (`text-white/40`)
- "전체 초기화" 텍스트만 짧게 → `초기화` + 아이콘
- 좌측 그룹에 `min-w-0` + 내부 버튼 그룹은 `shrink-0`으로 명시 → 절대 wrap 안 되게

### #2. 카드 hover 시 라벨 변경
`ProjectCard.tsx`: `Efficiency` → `진행률`로 변경.

### #3. D-day 옆 마감일 텍스트 강조
현재 `text-[11px] text-white/40` → 정보 위계가 D-day 배지보다 너무 낮음.
**수정**: `text-[12px] font-bold text-white/70` + 날짜 앞에 `Calendar` 아이콘(w-3 h-3) 추가 → "중요 정보" 위계 부여. "상시"인 경우는 텍스트만 표시.

### #4. Enter/Space 카드 열기
유지 (위 Q1 답변 참조).

### #5. FilterBar UI/UX 디자인 정리
- 라벨("부서", "상태") `text-[13px] text-white/40 font-semibold uppercase tracking-wider` — 보조 정보로 강등
- 버튼 내 count `(5)` → `font-mono text-[12px] text-white/40` (active 시 `text-white/70`) — 노이즈 감소
- Divider 높이 `h-6` → `h-8`, 색 `bg-white/15` — 그룹 분리감 강화
- "전체" 버튼에 컬러 닷 자리 시각 균형용으로 작은 회색 닷 또는 좌측 패딩 통일

### #6. 그 외 검토 결과 (Window A)
검토하면서 발견한 추가 개선점:

a. **정렬 버튼 그룹**: `최신순/진행률순/마감임박순`이 헤더 우측에 있는데 active 색상이 `bg-white/20`으로 선택 상태 대비 약함. → `bg-white/15` + 흰색 하단 바(2px) 추가로 active 상태 명확화. *우선순위 낮음 — 사용자 확인 후 진행.*

b. **빈 상태(Empty State)**: "조건에 맞는 프로젝트가 없습니다" 카드에 **현재 활성 필터를 보여주고 "필터 초기화" 버튼**을 직접 노출. 지금은 사용자가 상단까지 스크롤 올라가서 초기화해야 함.

c. **Header의 "Design Team" 로고 영역**: 사용자가 "로고 없음 — 일단 유지" 라고 했으므로 변경 없음.

d. **카드 grid `gap-y-16`**: hover scale 1.25를 위해 큰 여백. 의도된 것이므로 유지.

### #7. Window B (detail.tsx) 검토
b 화면도 한 번 살펴봤습니다:

a. **헤더의 진행률 바 색상**: `from-teal-500 to-emerald-500` 고정 — Window A에서 통일한 진행률 티어 색상(emerald/white/amber/red)과 다름. → **티어 시스템 동일 적용** 권장.

b. **"정말로 이 상세 업무를 삭제하시겠습니까?"** — 여전히 native `confirm` 사용 (Window A는 AlertDialog로 바꿨으나 detail에는 미적용). → AlertDialog로 통일.

c. **헤더 정보 영역**(마감일/PM/상태)이 한 줄에 너무 많이 들어가 폭이 큼. 1920px 기준은 OK지만 우측 버튼(전체 시안/시안 확대)과 충돌 가능. → 간격 조정.

d. **`<Image>` 아이콘 import**가 lucide-react에서 import되어 HTMLImageElement와 이름 충돌 가능. 현재는 동작하지만 잠재 이슈.
