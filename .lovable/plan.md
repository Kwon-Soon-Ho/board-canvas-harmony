# 대시보드 목적 적합성 검토 + 개선 계획

## 1. 즉시 수정: 부서/상태 카운트 가독성

### 현재 문제

지난 라운드에서 count(`(5)`)를 노이즈로 판단해 `text-[11px] text-white/30`으로 톤다운했지만 — 사용자 의도는 "버튼은 컴팩트, 숫자는 잘 보이게". 톤다운이 과했음.

### 수정안

숫자에 **배경 칩**을 둘러 별도 정보 단위로 격상:

```text
[● 영상  12]   ← 12는 흰색/60%, 옅은 배경 chip
[● 영상  12]   ← active일 때 흰색/100%, 진한 배경 chip
```

- 비활성: `bg-white/[0.06] text-white/60 font-bold text-[12px] font-mono`
- 활성: `bg-white/15 text-white font-bold text-[12px] font-mono`
- `px-1.5 py-0.5 rounded` chip 형태
- `tabular-nums`로 숫자 정렬

→ 버튼 자체의 컴팩트함은 유지하면서 숫자 가독성만 명확히 끌어올림.

대상 파일: `src/components/control/FilterBar.tsx` (부서/상태 두 곳)

## 2. 목적 적합성 종합 검토

### 대시보드 목적 (사용자 정의)

> 디자인팀의 프로젝트들을 **한눈에 파악**하고 **관리**

**한눈에 파악 = 스캐닝(scanning)** — 지금 무엇이 위험한지, 누가 무엇을 하는지, 어디가 막혔는지를 시각적으로 즉시 인지.
**관리 = 액션(action)** — 우선순위 결정, 자원 재배분, 위험 대응을 빠르게 실행.

### 지금 잘 작동하는 것 ✓

- 카드의 부서/상태 컬러 코딩 (시각 식별)
- 진행률 티어 색상 (emerald/white/amber/red) — 위험도 즉시 파악
- D-day 배지 + 마감일 이중 표시 (강화 후)
- 부서×상태 다차원 필터
- 정렬(최신/진행률/마감임박)

### 부족한 부분 — 우선순위별

#### 🔴 Critical (목적에 정면으로 미달)

**P1. 위험 신호 가시성 부재**

- 카드를 hover해야만 진행률·이슈가 보임. **기본 화면(non-hover)에서는 위험한 프로젝트를 식별 불가**.
- 카드에 hover 없이 보이는 위험 정보:
  - 진행률 바 (1-2px의 얇은 컬러 띠로 카드 상단 또는 하단에 상시 표시)
  - 활성 이슈 개수 배지 (이슈 ≥ 1 시 빨간 닷)
  - 마감 임박 글로우 (D-3 이내 시 카드 테두리에 미세한 amber/red 링)
- **"한눈에 파악" 미달의 가장 큰 원인.**

**P2. 종합 KPI 패널 없음**

- 헤더 아래에 한 줄짜리 "오늘의 상황" 요약이 없음. 매니저가 가장 먼저 보고 싶은 것:
  - 진행 중 N개 / 대기 N개 / 완료 N개
  - 마감 7일 이내 N개 (클릭 → 필터 적용)
  - 이슈 미해결 N개 (클릭 → 필터 적용)
  - 평균 진행률 X%
- 지금은 **"총 N개 프로젝트가 조건에 일치"** 한 줄뿐 — 정보량 부족.

**P3. 사람(담당자) 차원 부재**

- "디자인팀 관리"인데 **누가 과부하인지** 보이지 않음. 필터에 "담당자" 차원이 없고, 멤버별 부하/진행률 뷰도 없음.
- 최소: PM 필터 추가, 또는 멤버 워크로드 사이드 패널 (각 멤버 이름 + 담당 프로젝트 수 + 평균 진행률).

#### 🟡 Important (효율 저하)

**P4. 카드 hover scale 1.25의 부작용**

- 사용자가 "가려도 됨"으로 결정한 사항이지만 — 다른 카드 정보를 가리는 동안에는 **비교 스캔이 불가능**. 한 카드 hover 시 주변 3~5장의 정보를 동시에 못 봄.
- 대안 제안 (선택): hover 시 카드 **내부에서 expand**(스택 깊이만 증가, 폭은 그대로)하거나, hover 정보 패널을 **카드 옆 고정 영역**(우측 사이드바)에 표시.
- 사용자 결정 사항이므로 강제하지 않음. **단, "한눈에 파악" 목적과 충돌함을 명시.**

**P5. 정렬과 필터의 시각적 결합 약함**

- 현재 어떤 필터가 활성인지 보려면 상단 FilterBar를 봐야 하는데, 카드 grid 위 헤더("전체 프로젝트")는 이를 반영 안 함.
- 개선: H1을 "전체 프로젝트" → 동적으로 ("영상 부서 · 진행 중 12개" 등). 활성 필터를 칩으로 표시 + grid 영역 내에서 직접 제거 가능.

**P6. 마감/이슈 우선 뷰 없음**

- "마감임박순" 정렬은 있지만 **D-7 이내만** 보거나 **이슈가 있는 프로젝트만** 보는 빠른 토글이 없음.
- 헤더 KPI 칩(P2)이 직접 필터 토글이 되면 자연스럽게 해결.

#### 🟢 Nice to have (장기)

**P7. 보드 뷰(Kanban) 부재**

- 현재 Grid 단일 뷰. 상태(대기/진행/완료) 칼럼 보드 뷰가 있으면 워크플로우 관리 효율 ↑.
- 토글로 Grid ↔ Board 전환.

**P8. 시간축 뷰 부재**

- 마감일 기반 타임라인/캘린더가 없음. detail 화면에는 Gantt가 있는데 대시보드에는 부서 전체 타임라인이 없음.

**P9. 빈 영역 활용**

- 1920px 모니터 기준 max-w-[1600px]로 좌우 큰 여백. KPI 사이드바, 멤버 워크로드 패널 둘 곳 충분.

**P10. 활동 타임라인/최근 변경 사항**

- "오늘 무엇이 바뀌었는가" 피드가 없음. 출근 후 첫 진입 시 가장 알고 싶은 정보.

## 3. 권장 진행 순서

이번 메시지에서는 **#1 (count 가독성)**만 즉시 처리하고, 나머지 P1~P10은 이 plan을 통해 사용자가 우선순위를 골라주시는 방식을 권장.

### 빠른 효과 vs 작업량


| 항목                 | 효과         | 작업량   |
| ------------------ | ---------- | ----- |
| #1 count chip      | 중          | 5분    |
| P1 카드 상시 위험 신호     | **매우 큼**   | 30분   |
| P2 KPI 요약 바        | **매우 큼**   | 1-2시간 |
| P3 담당자 필터          | 큼          | 30분   |
| P5 동적 H1 + 활성 필터 칩 | 중          | 30분   |
| P6 마감/이슈 빠른 토글     | 큼 (P2와 묶음) | —     |
| P7 보드 뷰            | 큼          | 3-4시간 |
| P8 타임라인 뷰          | 큼          | 3-4시간 |
| P10 활동 피드          | 중          | 2시간   |


### 추천 로드맵 (작업량 대비 효과 최적)

**라운드 1 (이번 승인)**

- #1 count chip
- P1 카드 상시 위험 신호 (진행률 바 + 이슈 닷 + 마감 임박 글로우)

**라운드 2**

- P2 KPI 요약 바 (헤더 아래 한 줄)
- P5 동적 H1 + 활성 필터 칩
- P6 (P2와 동시 해결)

**라운드 3**

- P3 담당자 차원 (필터 + 워크로드 사이드 패널)

**라운드 4 (선택)**

- P7 보드 뷰 / P8 타임라인 뷰 / P10 활동 피드 중 사용자 우선순위
